Expand description

Storage-agnostic interface trait for bit-based output.

The encoder repeatedly generates arrays of code bits that are typically smaller than a byte (8 bits). A type implementing BitSink is used to arrange those bits typically in bytes, and transfer them to the backend storage. ByteSink (alias to MemSink<u8>) is a standard implementation of BitSink that stores code bits to a Vec of u8s.

Writes n-bits of zeros.

A default implementation using write_msbs is provided. An impl can provide a faster short-cut for writing zeros.
